# Остатки наборов — простое приложение

Одностраничное веб‑приложение для просмотра остатков проб/тестов по клиентам и наборам.
Всё приложение — это **один файл `index.html`** (библиотека для чтения Excel встроена прямо
в него), никакого сервера, установки или других файлов рядом не требуется.

## Как это работает

Поддерживаются два формата исходных Excel/CSV файлов лаборантов:

1. **Один файл на клиента** (как реально ведётся сейчас): в шапке листа — название лаборатории,
   строки — наборы, столбцы — даты операций, а над датами — строка‑пометка вида «ИФА Идаванг»,
   «Отправили в Белгород», «Приход набора», «Опыт». Приложение само находит по этим пометкам,
   какие наборы использовал каждый клиент (Идаванг, Тропарево и т.д.), и показывает по ним
   текущий общий остаток набора в лаборатории (столбец «Итого»). Служебные операции
   (приход, отправка на склад, внутренние опыты) клиентами не считаются.
2. **Плоская таблица** (столбцы «Клиент», «Набор», «Остаток» или «Кол‑во») — тоже поддерживается,
   на случай если понадобится.

Если структура файла не распознана автоматически — появится окошко, где можно вручную указать
имя клиента и нужные столбцы.

Данные сохраняются в браузере телефона (localStorage), поэтому при повторном открытии
ничего заново загружать не нужно — только когда появится обновлённый файл. Можно загружать
сразу несколько файлов за раз (по одному на клиента). Название клиента можно поправить (✎)
или удалить (🗑), если что‑то определилось неверно.

## Как открыть на телефоне

**Через ссылку (надёжнее всего):** разместите `index.html` на GitHub Pages или любом другом
статическом хостинге и откройте ссылку в браузере телефона, добавив её на главный экран.
Учтите: GitHub Pages на бесплатном тарифе работает только для **публичных** репозиториев —
для приватного репозитория либо сделайте его публичным, либо заведите отдельный публичный
репозиторий только для этого приложения.

**Локально (без хостинга):** можно просто переслать себе файл `index.html` и открыть его в
браузере телефона. Важно открыть именно **в браузере** (например, через «Поделиться» →
Safari/Chrome, или через меню браузера «Открыть файл»), а не через «Быстрый просмотр» в
приложении «Файлы» — режим предпросмотра не выполняет JavaScript и приложение не заработает.
